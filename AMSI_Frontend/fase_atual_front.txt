ROTAS CONFIGURADAS (App.jsx):
  /                  → LoginPage           (pública)
  /trocar-senha      → TrocarSenhaPage     (privada, sem navbar)
  /home              → HomePage            (privada + navbar)
  /usuarios          → UserListPage        (privada + navbar)
  /cliente_fornecedor → ClientRegisterPage (privada + navbar)
  /lancamento        → LancamentoPage      (privada + admin + navbar)
  /tipo_lancamento   → ListaLancamentosPage (privada + navbar)
  /cadastro          → UserRegisterPage    (privada + admin + navbar)

  Rotas protegidas usam Layout (Navbar + Outlet).
  Rotas admin usam AdminRoute dentro do Layout.

AUTH (services/auth.js):
  - Token armazenado em localStorage
  - getUserFromToken() — decodifica JWT (atob)
  - isAdmin() — verifica payload.perfil === "Administrador" (lê do JWT)
  - isAuthenticated() — valida token + expiração local
  - logout() — chama POST /auth/logout no backend antes de limpar localStorage

  ATENÇÃO: a expiração é validada apenas pelo payload JWT (exp).
  O header X-Session-Expires retornado pelo backend NÃO está sendo lido.

API (services/api.js):
  Totalmente alinhado ao contrato real do backend (openapi_ai.yaml com request_body
  e response_200 — gerado com função atualizada que resolve arrays e refs).
  Cobertura completa de endpoints:
  - Auth: login, logout, trocar-senha
  - Usuários: CRUD + resetar-senha
  - Cliente/Fornecedor: CRUD + resumo + filtros em getClifors
  - Endereço: CRUD + por-clifor
  - Contato: CRUD + por-clifor
  - Lançamento: CRUD + por-clifor + por-usuario + resumo (com filtros alinhados)
  - Tipo de Conta: CRUD
  - Login/Sessões: listagem + por-usuario

  fetchComLoading envolve todo fetch — aciona spinner global via loadingBus.
  handleResponse trata 401 com logout automático (exceto trocarSenha: noLogout: true).
  detail array do Pydantic tratado corretamente.

LOADING GLOBAL (services/LoadingContext.jsx):
  - LoadingProvider envolve o App inteiro
  - loadingBus singleton permite api.js acionar o contexto sem import React
  - Spinner overlay aparece durante qualquer requisição HTTP

LANÇAMENTO (pages/LancamentoPage.jsx):
  - Select de clifors e tipos de conta populados da API
  - Natureza derivada do tipo de conta selecionado (read-only)
  - Checkbox de reembolso inverte a natureza antes de enviar
  - Popup "+ Novo Tipo" cria tipo de conta inline, recarrega select e seleciona o novo
  - Somente admin acessa (/lancamento protegida por AdminRoute)

LISTA DE LANÇAMENTOS (pages/ListaLancamentosPage.jsx):
  - Carrega lançamentos ao montar, sem filtros
  - Filtros: clifor, tipo de conta, natureza, datas de vencimento/lançamento,
    apenas abertos, apenas vencidos, reembolso, valor mínimo/máximo
  - Badges de status: Aberto, Pago, Vencido, Estorno
  - Ação fechar: modal com data pagamento, valor pago, multa, juros, observação, reembolso
    (botão só aparece se lançamento ainda não foi pago e não é estorno)
  - Ação deletar: confirm() nativo + DELETE no backend
  - Nomes de clifor e tipo de conta resolvidos localmente (sem requisições extras)
