  /cadastro           → UserRegisterPage      (privada + admin + navbar)

AUTH (services/auth.js):
  - Token armazenado em localStorage('token')
  - Dados do usuário em localStorage('user') — populado no login via GET /usuarios/{id}
  - Expiração em localStorage('expiresAt') — timestamp Unix ms
  - getUserFromToken() — decodifica JWT (atob)
  - isAdmin() — verifica payload.perfil === "Administrador"
  - isConsulta() — verifica payload.perfil === "Consulta"
  - isAuthenticated() — valida token + expiração local
  - logout() — chama POST /auth/logout no backend antes de limpar localStorage

SESSÃO (App.jsx — MonitorSessao):
  - Verifica localStorage('token') e localStorage('expiresAt') a cada 10s
  - Se expirado: logout() + popup "Sessão expirada" + botão navega para /
  - Não dispara na tela de login (sem token = sem verificação)
  - X-Session-Expires lido em todo response autenticado (handleResponse no api.js)
    e salvo como timestamp ms em localStorage('expiresAt')
  - Backend envia X-Session-Expires como timestamp Unix ms (não ISO string)

API (services/api.js):
  - BASE_URL via import.meta.env.VITE_API_URL (AMSI_Frontend/.env)
  - Cobertura completa de endpoints
  - fetchComLoading envolve todo fetch — aciona spinner global
  - handleResponse trata 401 com logout automático (exceto trocarSenha: noLogout: true)
  - loginUser trata response sem body com .catch(() => ({}))
