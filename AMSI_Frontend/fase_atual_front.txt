ROTAS CONFIGURADAS (App.jsx):
  /                   → LoginPage             (pública)
  /trocar-senha       → TrocarSenhaPage       (privada, sem navbar)
  /home               → HomePage              (privada + navbar)
  /dashboard          → dashboard             (privada + admin + navbar)
  /tipo_lancamento    → ListaLancamentosPage  (privada + admin + navbar)
  /lancamento         → LancamentoPage        (privada + admin + navbar)
  /cliente_fornecedor → ClientRegisterPage    (privada + admin + navbar)
  /usuarios           → UserListPage          (privada + admin + navbar)
  /cadastro           → UserRegisterPage      (privada + admin + navbar)

COMPONENTES REUTILIZÁVEIS:

  ToastStack + useToast (components/ToastStack.jsx):
    - Notificações flutuantes no canto superior direito
    - mostrarToast(mensagem, tipo) — 'sucesso' | 'erro' | 'aviso'
    - mostrarToasts([msgs]) — escalonado: 1ª some em 5s, 2ª em 6s, 3ª em 7s...
    - Limpa toasts anteriores antes de disparar novo lote
    - Usado em: ClientRegister, UserList, UserRegister, LancamentoPage,
                ListaLancamentosPage, dashboard, PerfilCompletoPopup

  ModalConfirm (components/ModalConfirm.jsx):
    - CSS inline com variáveis do tema — funciona nos dois temas
    - Usado em: UserList (reset senha), PerfilCompletoPopup (associar/desvincular)

AUTH (services/auth.js):
  - Token armazenado em localStorage('token')
  - Dados do usuário em localStorage('user') — populado no login via GET /usuarios/{id}
  - Expiração em localStorage('expiresAt') — timestamp Unix ms
  - getUserFromToken(), isAdmin(), isConsulta(), isAuthenticated(), logout()

SESSÃO (App.jsx — MonitorSessao):
  - Verifica token/expiresAt a cada 10s
  - Se expirado: logout() + popup + navega para /
  - Não dispara na rota / (guard por useLocation)

TEMAS (styles/theme.css):
  - Verde (padrão): --primary #1B4332, --bg #EFE6DD
  - Corporativo: --primary #38BDF8, --bg #0F172A
  - Alternado via botão no topbar; salvo em localStorage('amsi_tema')
  - Todo CSS novo deve usar variáveis — nunca cores hardcoded

BACKEND — ENDPOINTS NOVOS (routes/usuario.py):
  GET    /usuarios/{id}/clifor                      — retorna clifor vinculado (404 se não houver)
  GET    /usuarios/{id}/clifor/sugestao?nome=...    — sugestões por similaridade (pg_trgm)
  POST   /usuarios/{id}/clifor/{id_clifor}/associar — vincula clifor ao usuário
  DELETE /usuarios/{id}/clifor/desvincular          — desvincula clifor do usuário

BANCO DE DADOS:

    após INSERT/UPDATE/DELETE em lancamento
    Regra: inadimplente = true se existir lançamento Crédito, não pago,
